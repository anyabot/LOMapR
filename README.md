# LO:MA Player Resource (lomapr)

A community database / reference site for the game, built with
[Next.js](https://nextjs.org/) (pages router) + [Chakra UI](https://chakra-ui.com/)
+ Redux Toolkit. It's a static-export SPA (`output: 'export'`) that fetches all
game data as JSON at runtime — units, enemies, skills, equipment, worlds/stages,
Sanctum, and Infinite War.

## Getting started

```bash
npm install
npm run dev          # dev server against remote data (R2)
npm run dev:local    # dev server against on-disk data in public/local-data/
```

Open [http://localhost:3000](http://localhost:3000).

| script | what it does |
|---|---|
| `dev` / `dev:bucket` | dev server; reads game data from the remote R2 bucket |
| `dev:local` | syncs `data/` → `public/local-data/`, then serves it statically |
| `sync:local-data` | copy `data/` → `public/local-data/` only |
| `build` / `pages:build` | static export (the latter also slices icons + cleans local data) |

## Pages

- **Units** (`/units`, `/units/detail`) — grid by class/role/grade, search by name
  or code; detail has stats calculator, skills, profile, drop locations, promotion,
  limit break, and exclusive equipment.
- **Equipment** (`/equipment`) — list filtered by type (Chip / OS / Item /
  Exclusive), grade, and exchange shop (Sanctum / Infinite War); search by name or
  code. A modal (`?equip=<id>` deep-link, shareable) shows per-rank stats/effects
  and the equip's Source tab (exchange shop + per-rank drop stages).
- **Enemies** (`/enemies`) — list + modal (`?enemy=<id>`) with stats, skills, AI.
- **World / Stages** (`/world`, `/world/stage`), **Sanctum** (`/sanctum`),
  **Infinite War** (`/iw`).

Equipment and enemy modals are mounted globally (`components/layout.tsx`), so a
drop chip or exclusive-gear tile anywhere can open them in place; unit references
get a shared hover-card (`components/unitHoverCard.tsx`).

## Data

The app reads JSON keyed per region (global / kr). Source is chosen by
`NEXT_PUBLIC_DATA_SOURCE`: remote R2 (default) or `local` (`public/local-data/`,
used by `dev:local`). See `lib/fetchData.ts`.

The JSON is generated by a local **data pipeline** under `tools/` — download +
extract game bundles → transform into `data/<region>/*.json`. That toolchain and
the generated `data/` are **gitignored** (local-only by policy; not part of a
clone), so a fresh checkout has no game data until the pipeline is run. The full
pipeline walkthrough lives in [`tools/README.md`](tools/README.md) (also untracked).

Icons are the exception: sprite PNGs are sliced once and committed under
`public/images/` (tracked).
